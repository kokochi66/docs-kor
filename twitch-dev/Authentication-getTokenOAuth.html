토큰 얻기 : OAuth 원본 사이트

소개

OAuth 처리 절차에는 3가지 방법이 있습니다.

  • OAuth 암시적 코드 흐름이 사용자 엑세스 토큰을 가져온다.
  • OAuth 인증 코드 흐름이 사용자 엑세스 토큰을 가져온다.
  • OAuth 클라이언트 자격 흐름이 사용자 엑세스 토큰을 가져온다.

자세한 다양한 타입과 절차에 대한 설명은 앱 과 인증 가이드를 확인하세요

OAuth 암시적 코드 흐름

1) 여러분이 인증하길 원하는 사용자의 정보를 등록된 리다이렉트 URI로 보냅니다. 인증 페이지에서는 사용자에게 트위치에 가입하거나, 로그인 하도록 요청하며, 사용자가 여러분의 어플리케이션의 시스템을 인증할 것인지 선택할 수 있게 도와줍니다.

<a href="">login</a> 를 생성하세요

1
2
3
4
5
GET https://id.twitch.tv/oauth2/authorize
    ?client_id=<your client ID>
    &redirect_uri=<your registered redirect URI>
    &response_type=<type>
    &scope=<space-separated list of scopes>
cs

요청에는 몇가지 필수 파라미터와 옵션 파라미터가 존재합니다.

요구 파라미터 타입 설명
client_id string

여러분의 클라이언트 ID 입니다.
redirect_uri URI

여러분의 등록된 리다이렉트 URI입니다. 이는 등록단계에서 등록된 것과 정확히 일치해야합니다.
response_type string

반환값으로 받을 정보를 정확하게 특정합니다. 엑세스 토큰의 형식으로 반환받고 싶다면, 이는 토큰 타입을 가져야 합니다. 사용자는 한번 혹은 여러번의 인증 확인을 해야 할 것입니다. 이는 아래의 옵션 파라미터인 force_verify에 의해 조절됩니다.
scope string

범위(scope) 리스트를 공백으로 구분하여 입력합니다.
옵션 파라미터 타입 설명
force_verify boolean

사용자에게 권한 부여에 대한 요청을 다시 요청해야 하는지를 체크합니다. 이것이 true 라면, 사용자는 항상 인증을 확인하라는 메시지가 나타나게 됩니다. 사용자는 API를 통해서 로그아웃을 할 방법이 없기 때문에, 사용자가 트위치 계정을 전환할 때 유용하게 사용할 수 있습니다. 기본값은 false 입니다.(false 상태라면, 사용자는 처음으로 지정된 범위에 대한 권한 부여를 하게 됩니다.)

state string

어플리케이션에서 생성 된 고유 토큰입니다. 이것은 CSRF 공격을 방지하기 위해서 사용되는 OAuth 2.0의 불투명한 값입니다. 이 값은 그대로 응답요청으로 반환됩니다. 우리는 이것을 사용하는 것을 강하게 추천드립니다.

다음은 viewing_activity_read 범위를 사용해서, 엑세스 요청을 보내고, 사용자를 http://localhost로 반환시키는 예제입니다.

1
GET 'https://id.twitch.tv/oauth2/authorize?response_type=token&client_id=uo6dggojyb8d6soh92zknwmi5ej1q2&redirect_uri=http://localhost&scope=viewing_activity_read&state=c3ab8aa609ea11e793ae92361f002671'
cs

2) 만약, 사용자가 여러분의 어플리케이션을 승인했다면, 사용자는 여러분의 리다이렉트 URL로 이동하게 됩니다.

1
https://<your registered redirect URI>#access_token=<an access token>
cs

엑세스 토큰은 쿼리 스트링이 아닌, URL 안에 들어있습니다. 즉, 여러분의 서버에 대한 HTTP 요청에 표시되지 않습니다. URI 객체는 document.location.hash를 자바스크립트 에서 실행시켜서 접근할 수 있습니다.

여러분이 요청시에 state 값을 넣었다면, 응답 객체에도 state값이 반환 됩니다.

다음은 예제입니다.

1
2
3
4
https://localhost#access_token=0123456789abcdefghijABCDEFGHIJ
    &scope=viewing_activity_read
    &state=c3ab8aa609ea11e793ae92361f002671
    &token_type=bearer

OAuth 인증 코드 흐름

1) 사용자에게 여러분의 등록된 리다이렉트 URI로 원하는 인증을 보내세요. 그러면 사용자에게 인증 페이지가 보내지고, 사용자는 트위치에 계정을 생성하거나, 로그인하여 여러분의 어플리케이션을 승인할지 여부를 선택할 것입니다. 다음 요청을 사용하세요.

1
2
3
4
5
GET https://id.twitch.tv/oauth2/authorize
    ?client_id=<your client ID>
    &redirect_uri=<your registered redirect URI>
    &response_type=code
    &scope=<space-separated list of scopes>
cs

다음은 요청에 들어가는 필수 파라미터와 옵션 파라미터입니다.

요구 파라미터 타입 설명
client_id string

여러분의 클라이언트 ID 입니다.
redirect_uri URI

여러분의 등록된 리다이렉트 URI입니다. 이는 등록단계에서 등록된 것과 정확히 일치해야합니다.
response_type string

이 값은 code 가 들어가야 합니다. 이 절차는 뒷부분에서 사용되는 인증 코드가 반환 되도록 하는 역할을 합니다. 사용자는 요청에 대해 승인을 확인하라는 메세지를 받게 됩니다.
scope string

범위(scope) 리스트를 공백으로 구분하여 입력합니다.

1
GET 'https://id.twitch.tv/oauth2/authorize?response_type=token&client_id=uo6dggojyb8d6soh92zknwmi5ej1q2&redirect_uri=http://localhost&scope=viewing_activity_read&state=c3ab8aa609ea11e793ae92361f002671'
cs

OAuth 클라이언트 자격 흐름

이전에 언급했 듯이, 앱 엑세스 토큰은 서버 간 API 요청에만 사용됩니다. 다음의 권한부여(grant) 요청에는 앱 엑세스 토큰을 얻기 위해서 클라이언트 시크릿이 필요합니다. 이는 또한 서버 간 요청에서만 사용되어야 하며, 클라이언트 코드에 노출되어서는 안됩니다.

1) 여러분의 서버에서, 다음 요청을 통해서 앱 엑세스 토큰을 얻습니다.

1
2
3
4
5
POST https://id.twitch.tv/oauth2/token
    ?client_id=<your client ID>
    &client_secret=<your client secret>
    &grant_type=client_credentials
    &scope=<space-separated list of scopes>
cs

다음은 예제 코드입니다.

1
POST https://id.twitch.tv/oauth2/token?client_id=uo6dggojyb8d6soh92zknwmi5ej1q2&client_secret=nyo51xcdrerl8z9m56w9w6wg&grant_type=client_credentials
cs

다음은 요청에 들어가는 필수 파라미터와 옵션 파라미터 입니다.

요구 파라미터 타입 설명
client_id string

여러분의 클라이언트 ID 입니다.
client_secret string

여러분의 클라이언트 시크릿 입니다.
grant_type string

클라이언트 자격(client_credentails)여야 합니다.
옵션 파라미터 타입 설명
scope boolean

공백으로 구분된 범위(scope)의 리스트입니다.
