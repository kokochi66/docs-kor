트위치 IRC : 커맨드(Commands) 원본 사이트

개요

이 가이드에서는 여러가지의 트위치 IRC 커맨드 기능에 대해서 설명합니다.

커맨드 설명
CLEARCHAT 사용자의 채팅 메세지를 제거합니다. 일반적으로 밴이나, 타임아웃 된 경우에 사용합니다.
CLEARMSG 사용자의 채팅 메세지 하나를 제거합니다. 이는 IRC상에서 /delete <target-msg-id> 트리거로 사용됩니다.
HOSTTARGET 채널에서 호스트 모드를 시작하거나 정지합니다.
NOTICE 서버로부터 나오는 공지사항 입니다.
RECONNECT 채널이 재시작된 후에 재접속 합니다.
ROOMSTATE 채널의 채팅 설정을 확인합니다.
USERNOTICE 채널에서 트위치의 특정 이벤트를 공지합니다.
USERSTATE 사용자의 채팅 설정이나 속성을 확인합니다.

CLEARCHAT (트위치 커맨드)

사용자의 채팅 메세지를 제거합니다. 일반적으로 밴이나 타임아웃 된 경우에 사용됩니다.

프로토타입

1
> :tmi.twitch.tv CLEARCHAT #<channel> :<user>
cs

이 기능은 추가적인 파라미터가 있는 CLEARCHAT(트위치 태그)기능과 함께 사용해야 합니다. 여러분이 IRC 봇에게 명령을 읽게 할 생각이라면, 태그 기능을 함께 사용해야 합니다. 대부분의 명령은 태그 없이는 유용하지 않거나 의미없게 사용될 수 있습니다. 추가적인 파라미터를 확인하려면, 트위치 태그 CLEARCHAT 정보를 확인하세요.

예제 1 : dallas 채널의 모든 채팅을 제거합니다.

1
> :tmi.twitch.tv CLEARCHAT #dallas
cs

예제 2 : ronni가 dallas채널에서 밴 당한 경우입니다.

1
> :tmi.twitch.tv CLEARCHAT #dallas :ronni
cs

CLEARMSG (트위치 커맨드)

사용자의 채팅 메세지 하나를 제거합니다. 이는 IRC상에서 /delete <target-msg-id> 트리거로 사용됩니다. 이 기능을 CLEARMSG 트위치 태그 기능과 함께 사용하세요.

프로토타입

1
> @login=<login>;target-msg-id=<target-msg-id> :tmi.twitch.tv CLEARMSG #<channel> :<message>
cs
파라미터 설명
login 메세지를 보낸 사용자의 이름
message 사용자가 보낸 메세지
target-msg-id 메세지의 UUID.

예제 : dallas 채널에서 ronnie의 HeyGuys라는 메세지가 삭제됩니다.

1
> @login=ronni;target-msg-id=abc-123-def :tmi.twitch.tv CLEARMSG #dallas :HeyGuys
cs

HOSTTARGET (트위치 커맨드)

채널의 호스트 모드를 시작하거나 중지시킵니다.

먼저 채널의 호스트모드를 시작하는 경우입니다.

프로토타입

1
> :tmi.twitch.tv HOSTTARGET #hosting_channel :<channel> [<number-of-viewers>]
cs

채널의 호스트모드를 정지하는 경우입니다.

프로토타입

1
> :tmi.twitch.tv HOSTTARGET #hosting_channel :- [<number-of-viewers>]
cs
파라미터 설명
number-of-viewers 시청하는 시청자의 수입니다. 사용하는 경우에는 양의 정수를 입력해야 합니다.

NOTICE (트위치 커맨드)

서버로부터 나오는 공지사항 입니다.

프로토타입

1
> @msg-id=<msg id>:tmi.twitch.tv NOTICE #<channel> :<message>
cs
파라미터 설명
message 사용자의 메세지입니다.
msg id 메세지의 ID 문자열입니다. i18ln으로 사용될 수 있습니다. 트위치 IRC msg-id 태그를 확인하세요

예제

1
> @msg-id=slow_off :tmi.twitch.tv NOTICE #dallas :This room is no longer in slow mode.
cs

RECONNECT (트위치 커맨드)

채널이 재시작 된 뒤에, 재접속 합니다.

가끔씩, 트우치 IRC의 프로세스가 재시작을 필요로 할 때가 있습니다. 이러한 일이 생길 때, IRC의 커맨드 기능은 RECONNECT 명령을 받게됩니다. 그리고 잠시 후에 연결이 끊어집니다. 이 경우에는 평소처럼 연결되어있던 채널을 재연결하고, 재접속해야 합니다.

ROOMSTATE (트위치 커맨드)

사용자가 채널에 참가하거나, 채팅방 설정을 할 때 사용합니다.

프로토타입

1
> :tmi.twitch.tv ROOMSTATE #<channel>
cs

IRC 태그 기능과 함꼐 사용하기 위해서, ROOMSTATE 트위치 태그의 추가적인 파라미터를 확인하세요

USERNOTICE (트위치 커맨드)

채널에서 트위치의 특정 이벤트를 공지합니다.

IRC 태그 기능과 함꼐 사용하기 위해서, USERNOTICE 트위치 태그의 추가적인 파라미터를 확인하세요

프로토타입

1
> :tmi.twitch.tv USERNOTICE #<channel> :message
cs

USERSTATE (트위치 커맨드)

사용자의 채팅 설정이나 속성을 확인합니다.

IRC 태그 기능과 함꼐 사용하기 위해서, USERSTATE 트위치 태그의 추가적인 파라미터를 확인하세요

프로토타입

1
> :tmi.twitch.tv USERSTATE #<channel>
cs
