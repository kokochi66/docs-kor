트위치 IRC : 태그 원본 사이트

개요

IRCv3 메시지 태그 특징에는 "메세지 태그는 메시지 별로 추가 메타 데이터를 추가하는 메커니즘 이다." 라고 명시되어 있습니다. 이는 기능 협상을 통해 활성화되는 프로토콜 메세지 포맷의 확장을 통해 이루어집니다. 태그는 선택적인(optional) 문자열을 값으로 갖는 간단한 키 입니다. 태그된 메세지는 서버와 클라이언트에게 보내질 수 있습니다. 각각의 태그가 어떻게 사용될지는 각각의 문서에 따라 달라집니다.

이와 반면에, 태그는 트위치가 IRC 메세지에 첨부할 수 있는 선택적인 메타데이터 이기도 합니다. 클라이언트가 태그기능을 요청하면, EBS에서, 한개 이상의 추가적인 태그를 클라이언트의 메세지에 추가합니다.

이 가이드 문서의 태그와 값에대한 내용은 트위치 IRC가 제공하는 분들입니다. 만약 현재 문서에서 태그가 존재하지 않는다면 해당 태그는 지원되지 않습니다.

다음은 태그 한 개에 해당하는 일반적인 포맷 예시입니다.

1
> @tag-name=<tag-name> :tmi.twitch.tv <command> #<channel> :<user>
cs

다음은 태그의 집합(set)에 해당하는 일반적인 포맷 예시입니다.

1
@tag-name-1=<tag-value-1>;tag-name-2=<tag-value-2>;... <rest of the command syntax depends on the particular IRC command used>
cs

태그는 무작위의 순서로 나타납니다. 여러분은 코드를 작성 할 때에, 코드를 하나하나 별도로 처리하도록 하고, 인식할 수 없는 태그는 무시하도록 작성하세요.

다음 표의 내용을 참고하세요. 몇가지 태그들은 더 이상 지원되지 않습니다.

커맨드 설명
CLEARCHAT 채널의 모든 메세지를 제거하거나, 특정 사용자가 밴되거나 타임아웃되었을 때에, 특정 사용자의 모든 메세지를 제거합니다.
CLEARMSG 채널에서부터 특정한 메세지 한 개를 제거합니다. 이는 IRC의 /delete <target-msg-id> 명령어를 통해서 실행됩니다.
GLOBALUSERSTATE IRC 태그를 이용해서 로그인을 성공한 유저에게 로그인된 유저의 정보를 제공합니다. 이는 인증이 성공된 이후에 PASS/NICK 커맨드를 통해서 보내집니다.
PRIVMSG 채널로 메세지를 보냅니다.
ROOMSTATE 사용자가 채널에 참여하였거나, 채널의 설정정보가 변경되었을 때, 채널의 상태정보를 보냅니다. 참가하였을 경우에는, 채팅방의 모든 설정정보가 포함되며, 설정정보가 변경된 경우에는 관련된 태그만이 보내집니다.
USERNOTICE 어떠한 이벤트가 발생했을 때, 사용자에게 알림을 보냅니다.
USERSTATE 사용자가 채널에 참가하였거나, PRIVMSG 명령어를 통해서 채널에 메세지를 보냈을 때, 사용자의 상태정보를 보냅니다.

CLEARCHAT (트위치 태그)

채널의 모든 메세지를 제거하거나, 특정 사용자가 밴되거나 타임아웃되었을 때에, 특정 사용자의 모든 메세지를 제거합니다.

프로토타입

1
 @ban-duration=<ban-duration> :tmi.twitch.tv CLEARCHAT #<channel> :<user>
cs
파라미터 설명
ban-duration (선택사항) 밴에 대한 기한입니다. 내용이 생략되면, 영구밴을 의미합니다.

예제 : ronni가 dallas 채널로부터 영구밴을 당한 경우

1
> :tmi.twitch.tv CLEARCHAT #dallas :ronni
cs

CLEARMSG (트위치 태그)

채널에서부터 특정한 메세지 한 개를 제거합니다. 이는 IRC의 /delete <target-msg-id> 명령어를 통해서 실행됩니다.

프로토타입

1
> @login=<login>;target-msg-id=<target-msg-id> :tmi.twitch.tv CLEARMSG #<channel> :<message>
cs
파라미터 설명
login 메세지를 보낸 사용자의 이름입니다.
message 메세지 입니다.
target-msg-id 메세지의 UUID 입니다.

예제 : ronni 의 HeyGuys라는 메세지가 dallas채널로부터 삭제됩니다.

1
> @login=ronni;target-msg-id=abc-123-def :tmi.twitch.tv CLEARMSG #dallas :HeyGuys
cs

GLOBALUSERSTATE (트위치 태그)

IRC 태그를 이용해서 로그인을 성공한 유저에게 로그인된 유저의 정보를 제공합니다. 이는 인증이 성공된 이후에 PASS/NICK 커맨드를 통해서 보내집니다.

프로토타입

1
> @badge-info=<badge-info>;badges=<badges>;color=<color>;display-name=<display-name>;emote-sets=<emote-sets>;turbo=<turbo>;user-id=<user-id>;user-type=<user-type> :tmi.twitch.tv GLOBALUSERSTATE
cs

참고 : 이를 사용하기 전에, 여러분은 요청을 보내고, 다음 태그를 받아야합니다.

1
2
< CAP REQ :twitch.tv/tags
> :tmi.twitch.tv CAP * ACK :twitch.tv/tags
cs

더 자세한 정보는 트위치 IRC 기능을 참조하세요

파라미터 설명
badge-info

badges 태그에 들어있는, 채팅 배지와 관련된 메타데이터 입니다.

현재 이 기능은 구독자가 해당 채널을 몇개월 구독했는지를 확인하기 위해서 구독자에게만 사용됩니다. 이에대한 수치는 배지의 버전의 수보다 훨씬 세밀화 되어 있습니다. 예를들어, 어떠한 사용자가 채널을 45개월동안 구독했다고 가정했을 때, badge-info를 이용하면 45개월이라는 값이 그대로 나오지만, badges version을 사용하면 오직 3년이라고만 표시됩니다.
badges 콤마(,)로 구분되어진 채팅 배지와 그 배지의 각각의 버전에 대한 정보가 담긴 리스트입니다.(각 데이터는 <badge>/<version>형식으로 포맷되어 있습니다.) badge의 값에는 여러가지 값이 존재합니다. (admin, bits, broadcaster, global_mod, moderator, subscriber, staff, turbo 등). 대부분의 배지들은 오직 버전 1만을 가집니다. 하지만 몇몇개의 배지들은 배지를 보유한 기간에 대비한 여러가지 버전이 존재합니다.
color 16진법으로 이루어진 RGB 색깔 코드입니다. 설정되지 않은 경우에는, 빈 문자열로 표시됩니다.
display-name IRC 스펙에 따라 표시되는 사용자의 이름입니다. 설정되지 않은 경우에는, 빈 문자열로 표시됩니다.
emote-sets 콤마(,)로 구분되어진 하나 이상의 이모티콘 세트로 이루어진 세트의 리스트 입니다. 이는 항상 최소값이 0으로 표시됩니다. 채팅 이모티콘을 집합에서 가져오기를 참고하시면 이모티콘의 하위집합을 확인할 수 있습니다.
turbo (현재는 사용되지 않습니다 badges로 대체되었습니다.) 사용자가 터보 배재를 갖고있으면 1, 없으면 0이 표시됩니다.
user-id 사용자의 ID입니다.
user-type (현재는 사용되지 않습니다. badges로 대체되었습니다.) 사용자의 타입입니다. empty, mod, global_mod, admin, staff의 값을 가집니다. 스트리머는 이중 어떠한 값도 가질 수 있습니다.

예제 : dallas의 로그인된 admin 사용자를 모두 조회합니다.

1
> @badge-info=subscriber/8;badges=subscriber/6;color=#0D4200;display-name=dallas;emote-sets=0,33,50,237,793,2126,3517,4578,5569,9400,10337,12239;turbo=0;user-id=1337;user-type=admin :tmi.twitch.tv GLOBALUSERSTATE
cs

PRIVMSG (트위치 태그)

채널로 메세지를 보냅니다.

프로토타입

1
> @badge-info=<badge-info>;badges=<badges>;color=<color>;display-name=<display-name>;emotes=<emotes>;id=<id-of-msg>;mod=<mod>;room-id=<room-id>;subscriber=<subscriber>;tmi-sent-ts=<timestamp>;turbo=<turbo>;user-id=<user-id>;user-type=<user-type> :<user>!<user>@<user>.tmi.twitch.tv PRIVMSG #<channel> :<message>
cs

모든 PRIVMSG 태그는 이전의 프로토타입 리스트가 포함됩니다. 또한 bits 필드는 비트 메세지용으로 전달됩니다. 이에 대한 더 자세한 정보를 알고싶으면, 확장 수익창출(Extensions Monetization) 가이드를 확인하세요

파라미터 설명
badge-info

badges 태그에 들어있는, 채팅 배지와 관련된 메타데이터 입니다.

현재 이 기능은 구독자가 해당 채널을 몇개월 구독했는지를 확인하기 위해서 구독자에게만 사용됩니다. 이에대한 수치는 배지의 버전의 수보다 훨씬 세밀화 되어 있습니다. 예를들어, 어떠한 사용자가 채널을 45개월동안 구독했다고 가정했을 때, badge-info를 이용하면 45개월이라는 값이 그대로 나오지만, badges version을 사용하면 오직 3년이라고만 표시됩니다.
badges 콤마(,)로 구분되어진 채팅 배지와 그 배지의 각각의 버전에 대한 정보가 담긴 리스트입니다.(각 데이터는 <badge>/<version>형식으로 포맷되어 있습니다.) badge의 값에는 여러가지 값이 존재합니다. (admin, bits, broadcaster, global_mod, moderator, subscriber, staff, turbo 등). 대부분의 배지들은 오직 버전 1만을 가집니다. 하지만 몇몇개의 배지들은 배지를 보유한 기간에 대비한 여러가지 버전이 존재합니다.
bits

비트메세지용 만으로 보내지는 것입니다. cheer/bits의 양은 사용자에 의해서 사용됩니다. 모든 인스턴스는 다음 정규식을 따릅니다.

1
/(^\|\s)<emote-name>\d+(\s\|$)/
cs

(여기서, <emote-name>란, Cheermote 얻기에서 반환된 이모티콘의 이름 값입니다.)

해당값을 적절한 이모티콘 값으로 대체해주어야 합니다.

1
static-cdn.jtvnw.net/bits/<theme>/<type>/<color>/<size>
cs
  • theme - light 또는 dark
  • type - animated 또는 static
  • color - red는 10000 비트 이상, blue는 5000-9999, green은 1000-4999, 보라는 100-999, 회색은 1-99
  • size - 1-4 사이의 숫자
color 16진법으로 이루어진 RGB 색깔 코드입니다. 설정되지 않은 경우에는, 빈 문자열로 표시됩니다.
display-name IRC 스펙에 따라 표시되는 사용자의 이름입니다. 설정되지 않은 경우에는, 빈 문자열로 표시됩니다.
emotes

이모티콘 이미지와 메세지 안의 대체 텍스트에 대한 정보입니다. 이는 널값일 수 있습니다. 문법은 다음과 같습니다.

1
<emote ID>:<first index>-<last index>,<another first index>-<another last index>/<another emote ID>:<first index>-<last index>...
cs
  • emote ID : 해당 URL을 사용하는 수 입니다. size값은 1.0, 2.0, 3.0 등이 올 수 있습니다.
    • 1
    http://static-cdn.jtvnw.net/emoticons/v1/:<emote ID>/:<size>
    		cs
  • first index, last index : 캐릭터의 위치값(index)입니다. \001ACTION 은 계산하지 않습니다. 사용자가 보낸 메시지의 첫번째 문자열부터 세기 시작합니다. 아래 예제를 확인하세요
id 메세지의 고유 id 입니다.
message 메세지 값을 의미합니다.
mod 사용자에게 관리자 배지가 있는 경우 1, 아니면 0입니다.
room-id 채널의 고유 id입니다.
subscriber (현재 사용하지 않음, 대신 badges를 사용함), 구독자 배지가 있는 경우 1, 아니면 0입니다.
tmi-sent-ts 서버가 메세지를 받았을 때의 타임스탬프 값 입니다.
turbo (현재는 사용되지 않습니다 badges로 대체되었습니다.) 사용자가 터보 배재를 갖고있으면 1, 없으면 0이 표시됩니다.
user-id 사용자의 ID입니다.
user-type (현재는 사용되지 않습니다. badges로 대체되었습니다.) 사용자의 타입입니다. empty, mod, global_mod, admin, staff의 값을 가집니다. 스트리머는 이중 어떠한 값도 가질 수 있습니다.

비트를 사용하지 않는 메세지 예제 : 첫번째 Kappa(emote ID 25)는 캐릭터값 0부터 캐릭터값 4까지 이다. 그리고 다른 Kappa는 12부터 16까지이다.

1
> @badge-info=;badges=global_mod/1,turbo/1;color=#0D4200;display-name=ronni;emotes=25:0-4,12-16/1902:6-10;id=b34ccfc7-4977-403a-8a94-33c6bac34fb8;mod=0;room-id=1337;subscriber=0;tmi-sent-ts=1507246572675;turbo=1;user-id=1337;user-type=global_mod :ronni!ronni@ronni.tmi.twitch.tv PRIVMSG #ronni :Kappa Keepo Kappa
cs

비트 메세지를 사용한 예제 :

1
> @badge-info=;badges=staff/1,bits/1000;bits=100;color=;display-name=ronni;emotes=;id=b34ccfc7-4977-403a-8a94-33c6bac34fb8;mod=0;room-id=1337;subscriber=0;tmi-sent-ts=1507246572675;turbo=1;user-id=1337;user-type=staff :ronni!ronni@ronni.tmi.twitch.tv PRIVMSG #ronni :cheer100
cs

ROOMSTATE (트위치 태그)

사용자가 채널에 참여하였거나, 채널의 설정정보가 변경되었을 때, 채널의 상태정보를 보냅니다. 참가하였을 경우에는, 채팅방의 모든 설정정보가 포함되며, 설정정보가 변경된 경우에는 관련된 태그만이 보내집니다.

프로토타입

1
> @emote-only=<emote-only>;followers-only=<followers-only>;r9k=<r9k>;slow=<slow>;subs-only=<subs-only> :tmi.twitch.tv ROOMSTATE #<channel>
cs
파라미터 설명
emote-only

이모티콘만을 사용하는 모드입니다. 활성화상태라면, 채팅에서 오직 이모티콘만이 허용됩니다. 0과 1 값을 가질 수 있습니다.
followers-only

팔로워만이 채팅이 가능한 모드입니다. 활성화상태라면, 팔로우 한 사용자만이 채팅이 가능합니다. 0과 1값을 가질 수 있거나, 음수가 아닌 정수값을 가질 수 있습니다.(지정 시간(분)동안 팔로우한 사용자만이 채팅이 가능합니다.)

r9k

R9k 모드입니다. 활성화상태라면, 9개 이상의 문자가 메세지 상에서 고유한 값을 가져야 합니다. 0과 1값을 가질 수 있습니다.
slow

관리자 권한이 없는 채팅들은 메세지가 보내질 때 까지 기다려야 합니다.
subs-only

구독자 만이 채팅이 가능한 모드입니다. 활성화상태라면, 구독자만이 채팅이 가능합니다 0과 1 값을 가질 수 있습니다.
